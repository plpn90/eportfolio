Cybersecurity

Secure software engineering plays a critical role in modern cybersecurity by ensuring the development of robust software systems capable of withstanding security breaches. This is achieved through the implementation of secure coding practices, comprehensive threat modelling, and rigorous testing methodologies.

Organisations such as the Open Web Application Security Project (OWASP) and the National Institute of Standards and Technology (NIST) contribute significantly to secure software engineering. OWASP’s "Top Ten" highlights prevalent web application security vulnerabilities, while NIST’s Cybersecurity Framework offers structured guidance on managing and mitigating these risks.

SQL injection is a serious cyber threat that allows attackers to hijack SQL queries and alter their execution behaviour. For instance, a harmless query to retrieve product details could be maliciously modified to delete data. Some attacks can be mitigated by granting minimum permissions, but others, such as returning extra details from a SELECT query, are harder to detect.

Following is an example of an SQL injection. The query would normally retrieve details for a specific user, but by specifying a username that resembles a tautological SQL statement, the query leaks details of all users. This can cause significant privacy breaches and reveal sensitive information like credit card details. Leaked users could also contain administrator credentials, adding further complications.

To mitigate such vulnerabilities, user input should be sanitised and special characters - such as those with special meaning to SQL - must be escaped, as escaped characters are rendered useless for logical alterations.
